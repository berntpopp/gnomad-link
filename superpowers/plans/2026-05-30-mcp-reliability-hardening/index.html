    L-4 follow-through (post-review) — population shaping unified

    The first pass shipped L-4 for get_gene_summary only. A re-review found the same untrimmed-population firehose still live in the other two tools the finding named:

    • get_variant_details compact never trimmed exome/genome populations — F508del returned the full 28.8 kB payload under the advertised "compact ~3 kB" (200+ HGDP/1kg/sex-split/zero-AC rows).
    • get_gene_variants passed every variant's full per-population breakdown through raw.

    Fix (cohesive split, mirrors the get_variant_frequencies defaults):

    • [x] Extract the population projector into gnomad_link/mcp/population_shaping.py (filter_populations, build_populations_truncated, project_variant_source, population_projection_note). Drops shaping.py 575 → 555, off its LOC ceiling.
    • [x] shape_variant_details_compact trims exome/genome populations + adds populations/include_subcohorts/include_sex_split/exclude_zero_populations toggles; per-source truncated.kind == "populations". Live: 28.8 kB → 4.8 kB (83%).
    • [x] shape_gene_variants trims each variant's populations + include_populations scan mode; one payload-level population_projection note (not per-row blocks). Live: include_populations=False ~30% leaner.
    • [x] Token hints corrected in tool docstrings + resources.py; limitations note widened to all three variant-bearing tools.
    • [x] TDD: 16 new unit tests (shaping + SDK-handler wiring/output-schema) + 3 live integration assertions. make ci-local green (format, lint, lint-loc, mypy, 393 tests).
